# Portfolio News Monitor

A Claude Code plugin that scaffolds a daily portfolio-news intelligence dashboard. Designed for VCs (or anyone tracking a list of companies) who want a single page that surfaces what's new across their portfolio every morning, with valuation-sensitive items flagged at the top.

## What it does

One skill — `portfolio-news-setup` — that interviews you for your portfolio, drops a customizable methodology and dashboard template into a project folder, and creates two scheduled tasks:

- **`portfolio-news-daily`** — runs every weekday morning. Searches the web for new news, podcast appearances, and video coverage per company. Classifies each item as Hot (valuation-sensitive), Watch (useful context), or Noise. Decays Hot to Watch after three days. Rewrites the dashboard.
- **`portfolio-news-weekly`** — runs Friday afternoon. Produces a digest of the week's Hot items with portfolio read-throughs.

After setup, the plugin's job is done. Everything you might customize lives in your workspace, not in the plugin.

## Installation

In Claude Code, add the marketplace and install the plugin:

```
/plugin marketplace add camrail/defiant-lp-claude-skills
/plugin install portfolio-news-monitor@defiant-lp-claude-skills
```

Then invoke the skill:

> Set up a portfolio news monitor.

The skill will ask for your portfolio companies, scaffold the workspace files, and create the scheduled tasks.

## What gets written to your workspace

In the project folder you choose (e.g. `~/Documents/Claude/Projects/Portfolio News Monitor`):

| File | Purpose | Edit to change |
|---|---|---|
| `companies.json` | Your portfolio list | Add or remove companies |
| `methodology.md` | Triage rules, Hot/Watch/Noise criteria, decay window, dashboard structure | Change classification logic or section layout |
| `dashboard-template.html` | Styled HTML shell with content regions marked by comments | Restyle the dashboard, swap colours, reorganize sections |
| `.state/seen-items.json` | Every item ever surfaced (initialized empty) | Hand-curate the history, if you ever need to |
| `dashboard.html` | Generated by each daily refresh | Don't edit — it's rewritten every morning |

The daily and weekly scheduled tasks reference these files by reading them at run time. That means edits take effect on the next run — no plugin update needed.

| Component | Type | Description |

## Requirements

- Claude Code with the **scheduled-tasks** MCP connector enabled (built-in)
- Web search available (built-in)

No additional connectors required.

## Customization

This plugin is intentionally a one-shot scaffolder. After setup, you own the workspace files and the running scheduled tasks. To change anything:

- **Add or remove companies** → edit `companies.json`
- **Change what counts as Hot / Watch / Noise, or the decay window** → edit `methodology.md`
- **Restyle the dashboard** → edit `dashboard-template.html` (CSS at top, structure in body)
- **Change the schedule** → edit the scheduled tasks in Claude Code's Scheduled Tasks UI
- **Pause everything** → disable the scheduled tasks

To re-run setup against a different folder (e.g. for a second portfolio), invoke the skill again and point it at a different project directory.
